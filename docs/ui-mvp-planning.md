# 타로맨스 메뉴 앱 UI MVP 기획문서

작성일: 2026-05-11  
대상: 현재 React/Vite MVP 앱 UI 개선  
전제: 기존 기능 MVP와 데이터 fallback 구조는 유지한다.

## 1. UI MVP 목표

현재 앱은 기능 흐름과 데이터 구조를 검증하기 위한 MVP 셸이다. 다음 UI MVP의 목표는 기능을 늘리는 것이 아니라, 매장 태블릿에서 손님이 봤을 때 “타로맨스에 접속했다”는 인상과 메뉴판으로서의 가독성을 동시에 확보하는 것이다.

핵심 목표:

- 기존 컨셉인 게임 접속, 매직 스크롤, 타로카드 선택 경험을 강화한다.
- 메뉴판으로서 읽기 쉬운 글자 크기, 대비, 여백, 터치 영역을 확보한다.
- 가로/세로 태블릿 UI 구조는 유지한다.
- 불필요한 설명형 UI는 추가하지 않는다.
- 기능 로직, 데이터 구조, Firebase fallback 흐름은 건드리지 않는다.

## 2. 유지할 UI 원칙

유지:

- 포털 화면에서 눌러 메뉴 앱으로 진입한다.
- 메뉴 진입 후 기본 화면은 가이드다.
- 메뉴 화면에 초기화면 복귀 버튼은 두지 않는다.
- 언어는 우측 상단 `[언어]` 버튼으로 선택한다.
- 가로 화면은 좌측 카테고리, 우측 스크롤 메뉴 구조를 유지한다.
- 세로 화면은 하단 카테고리, 상단 스크롤 메뉴 구조를 유지한다.
- 타로 시그니처 칵테일은 리스트가 아니라 카드 선택 UI로 제공한다.
- 타로 카드 비율은 7:12를 유지한다.
- 품절 메뉴는 숨기지 않고 붉은 이중 취소선과 `SOLD OUT`으로 표시한다.

바꾸지 않을 것:

- 카테고리/탭 구성
- 데이터 번들 구조
- Firebase 읽기 fallback 구조
- 메뉴 정렬 기준 `sort_code`
- 가격 축약 표시 방식

## 3. 현재 UI 문제

포털 화면:

- 현재 포털은 동작 확인용에 가깝고, 게임 접속 화면의 긴장감이 약하다.
- 중앙 진입 버튼이 카드처럼 보이지만, “입장 게이트” 느낌은 더 강화할 수 있다.
- 데이터 상태 문구는 필요하지만 시각적으로 덜 튀게 정리해야 한다.

카테고리 버튼:

- 버튼이 기능적으로는 명확하지만, 매장 세계관의 이미지 버튼 느낌이 약하다.
- 아이콘과 텍스트는 유지하되 버튼마다 문양/재질감을 조금 더 줄 필요가 있다.
- 활성 카테고리와 비활성 카테고리의 대비는 좋지만 다소 평면적이다.

스크롤 패널:

- 양피지 배경은 잘 보이지만 내부 메뉴 UI가 아직 일반 앱 카드 느낌이다.
- 상단 여백과 언어 버튼 위치가 빈 공간처럼 느껴질 수 있다.
- 공지 한 줄 영역은 존재하지만 매직 스크롤 속 문장처럼 보이도록 다듬을 수 있다.

메뉴 리스트:

- 글자와 가격 정보는 읽히지만, 실제 메뉴판의 고급스러운 밀도는 부족하다.
- 이미지 없는 메뉴의 fallback 상태가 충분히 브랜드화되어 있지 않다.
- 품절 상태는 기능적으로 보이지만 SOLD OUT 시각 레이어를 더 명확히 할 수 있다.

타로 카드:

- 7:12 비율과 캐러셀 구조는 구현되어 있다.
- 현재 카드는 임시 배경이라 “캐릭터 선택창” 집중도가 약하다.
- 중앙 카드, 좌우 카드, 버튼, 가격 정보의 위계가 더 필요하다.

상세 모달:

- 상세 정보를 열 수 있지만, 메뉴판의 최종 선택 화면으로서 고급감이 부족하다.
- 이미지, 잔 아이콘, 이름, 가격, 설명의 정보 위계를 더 명확히 해야 한다.
- 품절 상세 상태도 더 명확해야 한다.

## 4. 화면별 UI MVP 요구사항

### 4.1 포털 화면

목표:

- 손님이 “게임에 접속한다”는 느낌을 받는다.
- 터치해야 할 대상이 즉시 명확하다.
- 데이터 상태 문구는 작고 차분하게 보인다.

작업:

- 포털 중앙 게이트를 더 입체적인 버튼으로 개선한다.
- `TAROMANCE` 타이틀을 브랜드 시그널로 강화한다.
- 로딩/데이터 상태 문구는 게이트 하단 보조 정보로 낮춘다.
- 배경 이미지는 기존 `iv_mainbg.jpg`를 유지하되 어두운 오버레이를 조정한다.
- 언어 버튼은 우측 상단에 유지한다.

완료 기준:

- 첫 화면에서 브랜드명과 진입 버튼이 첫눈에 보인다.
- 버튼 텍스트가 12.7인치 태블릿 가로/세로에서 잘리지 않는다.
- 포털 진입 버튼의 터치 영역이 충분히 크다.

### 4.2 카테고리 버튼

목표:

- 카테고리 버튼이 단순 내비게이션이 아니라 게임 메뉴 버튼처럼 보인다.
- 가로/세로 배치가 모두 안정적이다.

작업:

- 버튼 배경에 미세한 재질감, 테두리, 활성 광택을 추가한다.
- 아이콘 크기와 텍스트 줄바꿈 기준을 정리한다.
- `와인 & 기타 주류`처럼 긴 텍스트도 세로 하단 버튼에서 깨지지 않게 한다.
- 활성 버튼은 금색만으로 채우기보다 테두리/명암/아이콘 대비까지 조정한다.

완료 기준:

- 네 카테고리 모두 동일한 크기 체계를 유지한다.
- 가로에서 좌측 버튼, 세로에서 하단 버튼이 흔들리지 않는다.
- 긴 언어 텍스트에서도 버튼 레이아웃이 깨지지 않는다.

### 4.3 스크롤 패널

목표:

- 메뉴 영역이 “마법 스크롤을 펼친 화면”처럼 보인다.
- 메뉴 정보는 장식보다 우선해서 읽힌다.

작업:

- 패널 내부 여백을 태블릿 기준으로 재조정한다.
- 상단 언어 버튼 영역이 떠 보이지 않도록 정렬한다.
- 탭 버튼과 공지 영역을 스크롤 종이 위 UI처럼 정리한다.
- 스크롤 패널의 배경 대비를 조절해 글자 가독성을 높인다.

완료 기준:

- 메뉴 이름, 가격, SOLD OUT 표시가 한눈에 읽힌다.
- 탭과 공지 영역이 서로 겹치지 않는다.
- 세로 화면에서도 스크롤 가능한 영역이 충분하다.

### 4.4 메뉴 리스트

목표:

- 일반 메뉴는 빠르게 훑어볼 수 있고, 가격과 품절 상태가 즉시 보인다.

작업:

- 메뉴 아이템을 반복 카드로 유지하되, 안쪽 정보 위계를 강화한다.
- 잔 이미지 또는 fallback 아이콘의 크기와 위치를 통일한다.
- 가격 숫자는 메뉴명보다 작지만 명확하게 보이도록 조정한다.
- 품절 메뉴는 메뉴명 이중 취소선, SOLD OUT 라벨, 낮은 대비를 함께 적용한다.

완료 기준:

- 메뉴명, 요약, 가격, 품절 상태가 겹치지 않는다.
- `16` 같은 축약 가격이 가격 정보로 명확하게 인식된다.
- 품절 메뉴도 선택 가능한 상태로 보이되 주문 가능 메뉴와 구분된다.

### 4.5 타로 카드 캐러셀

목표:

- 타로 시그니처 칵테일 탭은 앱의 대표 경험으로 느껴져야 한다.
- 손님이 좌우 이동과 카드 선택을 즉시 이해한다.

작업:

- 중앙 카드를 더 크게, 좌우 카드를 더 명확히 뒤로 배치한다.
- 좌우 화살표 버튼을 스크롤 컨셉에 맞게 보완한다.
- 카드 번호, 이름, 가격의 위치와 크기를 재정리한다.
- SOLD OUT 카드의 상태를 더 분명하게 표시한다.
- 실제 카드 이미지가 들어오기 전까지 임시 카드 배경이 7:12를 안정적으로 유지하게 한다.

완료 기준:

- 중앙 카드가 현재 선택 대상임을 즉시 알 수 있다.
- 좌우 버튼 클릭과 스와이프가 기존 기능대로 동작한다.
- 10장 카드가 유지되고 상세 모달 진입이 깨지지 않는다.

### 4.6 상세 모달

목표:

- 메뉴 선택 후 상세 화면이 “최종 확인”처럼 고급스럽고 명확해야 한다.

작업:

- 이미지 영역과 정보 영역의 비율을 정리한다.
- 메뉴명, 종류, 가격, 설명의 위계를 강화한다.
- 닫기 버튼은 명확하되 과하게 튀지 않게 한다.
- 품절 상세 상태는 상단 또는 가격 영역에서 분명히 보이게 한다.

완료 기준:

- 상세 모달이 가로/세로에서 화면 밖으로 넘치지 않는다.
- 긴 다국어 문구가 버튼/영역 밖으로 튀지 않는다.
- 닫기 동작과 바깥 클릭 닫기가 유지된다.

## 5. 디자인 자산 정책

현재 사용 가능 자산:

- `iv_loading.png`
- `iv_mainbg.jpg`
- `iv_scroll_1920h_03.png`
- `iv_popup_background.png`
- `noimage.png`
- `arrow.png`
- glass 이미지 6종

UI MVP에서 새로 필요한 자산:

- 포털 배경 최종 이미지 또는 영상
- 카테고리 버튼용 이미지/문양
- 타로 카드 임시/fallback 이미지 7:12
- SOLD OUT 이미지 또는 라벨 스타일
- 메뉴 이미지 fallback
- 상세 모달 배경 또는 프레임

이번 UI MVP에서는 새 이미지 제작 없이 기존 자산과 CSS만으로 1차 개선한다. 새 이미지가 필요하면 별도 자산 제작 MVP로 분리한다.

## 6. 색상/타이포 기준

방향:

- 어두운 잉크색, 양피지색, 금색, 붉은 품절색, 청록 포인트를 사용한다.
- 한 가지 색 계열만 반복하지 않는다.
- 과한 보라/파랑 그라데이션이나 장식용 빛 구체는 사용하지 않는다.
- 메뉴 가독성이 장식보다 우선이다.

타입:

- 브랜드/타로 카드 제목: display serif
- 메뉴명/버튼/가격: system sans
- 버튼 글자는 작아도 터치 영역은 크게 유지한다.
- viewport 기준으로 글자 크기를 과하게 스케일하지 않는다.

## 7. 반응형 기준

대상 태블릿:

- Lenovo Xiaoxin Pad 2024 12.7-inch
- 16:10 비율
- 고해상도 태블릿이지만 CSS 기준은 브라우저 viewport로 검증한다.

검증 뷰포트:

- Landscape: 1472 x 920
- Portrait: 920 x 1472
- Desktop fallback: 1920 x 1080

필수 확인:

- 카테고리 버튼 위치
- 언어 메뉴 위치
- 탭/공지/메뉴 리스트 겹침 여부
- 타로 카드 중앙 정렬
- 상세 모달 넘침 여부

## 8. UI MVP 작업 순서

1. CSS 토큰 정리
2. 포털 화면 개선
3. 카테고리 버튼 개선
4. 스크롤 패널과 탭/공지 정리
5. 메뉴 리스트 개선
6. 타로 카드 캐러셀 개선
7. 상세 모달 개선
8. 가로/세로 브라우저 QA
9. UI MVP QA 문서 작성

## 9. QA 체크리스트

정적 검증:

- `npm run lint`
- `npm run build`

브라우저 QA:

- 포털 화면이 브랜드 중심으로 보인다.
- 포털 진입 버튼이 명확하다.
- 메뉴 진입 후 가이드가 기본으로 보인다.
- 언어 메뉴가 우측 상단에서 잘 열린다.
- 네 개 카테고리 버튼이 가로/세로에서 깨지지 않는다.
- 칵테일의 Custom Cocktail 탭은 숨김 상태를 유지한다.
- 타로 카드 10장이 렌더링된다.
- 타로 좌우 버튼과 스와이프가 동작한다.
- 상세 모달이 열리고 닫힌다.
- 품절 메뉴의 이중 취소선과 SOLD OUT이 보인다.
- 와인 & 기타 주류 전체 탭이 보인다.
- 콘솔 에러가 없다.

시각 QA:

- 텍스트가 버튼/카드/모달 밖으로 튀지 않는다.
- 메뉴명, 가격, 품절 상태가 겹치지 않는다.
- 카드와 패널이 화면 밖으로 밀리지 않는다.
- 가로/세로 모두 손가락으로 누르기 쉬운 크기다.

## 10. 완료 기준

UI MVP는 다음 조건을 만족하면 완료로 본다.

- 기존 기능 QA 22개 항목을 유지한다.
- 가로/세로 주요 화면 스크린샷에서 UI 겹침이 없다.
- 포털, 메뉴, 타로 카드, 상세 모달이 현재보다 명확하게 타로맨스 컨셉을 전달한다.
- 새 기능 추가 없이 디자인 개선만으로 완료한다.
- UI MVP QA 문서와 커밋이 남아 있다.
