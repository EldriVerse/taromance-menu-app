# 세로모드 UI QA

작성일: 2026-05-14

## 기준

- 태블릿 원본 해상도 기준: 1840 x 2944
- 확인 화면: 초기 진입 후 가이드 카테고리 / 이용안내 탭

## 확인 항목

- 리스트 아이템 폭: 스크롤 배경의 좌우 찢어진 가장자리 안쪽으로 들어오도록 보정.
- 세부 탭 위치와 크기: 상단 말린 스크롤 영역 안에서 터치 영역을 확대.
- 언어 버튼 위치: 우측 상단 가장자리에서 안쪽으로 이동하고 터치 영역 확대.
- 공지 티커: 리스트와 같은 본문 폭 기준으로 정렬.
- 팝업창 크기: 세로모드에서 폭과 높이를 확대하되 이미지가 본문을 밀어내지 않도록 이미지 영역 비율 조정.
- 팝업 이미지: `cover` 대신 `contain`으로 변경해 임시 이미지나 세로형 이미지를 잘라내지 않도록 보정.

## 검증

- `npm run build` 통과.
- `npm run lint` 통과.
- Codex in-app browser에서 세로 해상도 기준으로 가이드 화면과 메뉴 상세 팝업을 확인.
